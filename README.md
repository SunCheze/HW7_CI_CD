# HW7_CI_CD
Задание выполняется в рамках модуля 7 «Автоматизированное развертывание с помощью CI/CD».

## CI/CD Pipeline для ML-сервиса
Обоснование стратегии деплоя
В рамках проекта рассматривались стратегии Blue-Green, Canary и Shadow. Была выбрана концепция Shadow (теневого) развертывания как базовая для тестирования, так как в исходном коде ml_pipeline.py полностью отсутствует обработка ошибок (try-except). Если использовать Canary-релиз, некорректные данные на проде вызовут краш сервиса у 5-10% реальных пользователей. При Shadow-деплое дубликаты запросов идут в тестовую модель, и её падение не влияет на реальных пользователей. После добавления обработки ошибок стратегию можно сменить на Canary.

## Структура проекта
ml_pipeline.py — скрипт обучения модели.
requirements.txt — фиксированные версии библиотек.
Dockerfile — описание контейнера.
docker-compose.blue.yml — конфигурация синего (стабильного) окружения.
.github/workflows/ci.yml — автоматизированный CI/CD пайплайн.

## Шаги запуска проекта
Локальный запуск через Docker Compose
Клонируйте репозиторий.
Находясь в корне проекта, выполните команду:
```docker compose -f docker-compose.blue.yml up --build```
В терминале появится результат работы ML-модели.
Запуск через CI/CD (GitHub Actions)
Сделайте git push в ветку main.
Перейдите в репозиторий на GitHub во вкладку Actions.
Пайплайн запустится автоматически. В логах шага Make pipeline reproducible вы увидите прогон скрипта, а шаг Deploy Blue Environment успешно развернет контейнер на runner'е GitHub.
